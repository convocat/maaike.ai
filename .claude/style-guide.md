# Style guide

Rules for all content Claude generates or helps write for this garden. Non-negotiable.

## Language

- **Never use em-dashes** (—). Use colons, commas, or periods instead.
- **Sentence case for titles.** Only capitalise the first word and proper nouns/acronyms.
- Content language is English. Maaike works in Dutch and English.

## Voice and attribution

- Refer to yourself as "Claude", never "AI" or "I"
- Never use "write" for Claude's output. Use "generate"
- Only Maaike writes. Claude generates.

## Content rules

- No filler phrases ("certainly", "of course", "absolutely", "great question")
- Keep descriptions concise: one sentence, no trailing period
- Tags: lowercase, hyphenated, specific. Never `ai` as a standalone tag.
- Frontmatter: always sentence case for `title`, never leave `description` empty before publishing

## Punctuation reminders

- Em-dash (—): NEVER. Replace with colon or split into two sentences.
- En-dash (–): only for ranges (e.g. 2020–2024)
- Ellipsis: use sparingly, never to trail off in titles
